# Contributing to OSINT Workshop

Thank you for your interest in contributing to the OSINT Workshop project! We welcome contributions from the community.

## How to Contribute

### Reporting Issues

If you find a bug or have a suggestion:

1. **Check existing issues** to avoid duplicates
2. **Create a new issue** with a clear title and description
3. **Include details:**
   - Steps to reproduce (for bugs)
   - Expected vs actual behavior
   - Browser/OS information
   - Screenshots if applicable

### Submitting Changes

1. **Fork the repository**
   ```bash
   git clone https://github.com/NomanMunir/OSINT-Workshop.git
   cd OSINT-Workshop
   ```

2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make your changes**
   - Follow the existing code style
   - Keep commits focused and atomic
   - Write clear commit messages

4. **Test your changes**
   - Open all HTML files in multiple browsers
   - Test responsive design on different screen sizes
   - Verify all interactive features work

5. **Commit your changes**
   ```bash
   git add .
   git commit -m "feat: add new activity for email OSINT"
   ```

6. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

7. **Open a Pull Request**
   - Describe what changes you made
   - Reference any related issues
   - Wait for review and feedback

## Contribution Guidelines

### Code Style

- **HTML:** Use semantic HTML5 tags, proper indentation (2 or 4 spaces)
- **CSS:** Use Tailwind utility classes, maintain consistent spacing
- **JavaScript:** Use modern ES6+ syntax, clear function names
- **Comments:** Add comments for complex logic only

### Activity Design Principles

When creating new activities:

1. **Beginner-Friendly:** Assume no prior OSINT knowledge
2. **Clear Objectives:** Each activity should have a specific learning goal
3. **Step-by-Step:** Provide detailed instructions
4. **Ethical Focus:** Include legal and ethical warnings
5. **Interactive:** Include hands-on challenges
6. **Visual Appeal:** Maintain the cyber intelligence theme

### Commit Message Format

Use conventional commits:

- `feat:` New features or activities
- `fix:` Bug fixes
- `docs:` Documentation updates
- `style:` UI/styling changes
- `refactor:` Code restructuring
- `test:` Adding tests
- `chore:` Maintenance tasks

Examples:
```
feat: add social media OSINT activity
fix: correct copy button functionality in activity 2
docs: update README with new activity details
style: improve mobile responsiveness
```

## Content Guidelines

### Educational Content

- Ensure all techniques are **legal** and **ethical**
- Provide **clear warnings** about legal boundaries
- Use **real, safe examples** (public datasets, authorized sites)
- Avoid **sensitive or controversial** targets

### Example Quality

- Use **publicly accessible** examples only
- Provide **alternative examples** if primary ones fail
- Keep examples **relevant** and **up-to-date**
- Ensure examples work **internationally**

### Security Considerations

- Never include **actual API keys** or credentials
- Use **placeholder** URLs for submission forms
- Warn against **unauthorized access**
- Promote **responsible disclosure**
